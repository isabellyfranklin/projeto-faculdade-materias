Servidores DHCP e DNS

O DHCP (Dynamic Host Configuration Protocol) e o DNS (Domain Name System) são dois servidores essenciais para o funcionamento de uma rede. O DHCP distribui endereços IP automaticamente para os dispositivos, enquanto o DNS traduz nomes de domínio em endereços IP.

DHCP — Dynamic Host Configuration Protocol

Protocolo responsável por atribuir endereços IP automaticamente a dispositivos conectados à rede, sem necessidade de configuração manual.

Além do IP, o DHCP também fornece:

  • Máscara de sub-rede
  • Gateway padrão
  • Endereço do servidor DNS

Funciona no modelo cliente–servidor: o cliente solicita e o servidor concede. Utiliza as portas UDP 67 (servidor) e UDP 68 (cliente).

1. Processo DORA

A comunicação entre cliente e servidor DHCP segue quatro etapas:

  1. Discover — o cliente envia um broadcast procurando um servidor DHCP na rede
  2. Offer — o servidor responde oferecendo um IP disponível
  3. Request — o cliente solicita formalmente o IP oferecido
  4. Acknowledge — o servidor confirma e registra a concessão (lease)

2. Pool de Endereços

O servidor DHCP possui um intervalo de IPs disponíveis para distribuição, chamado de pool. O administrador define quais IPs fazem parte desse intervalo.

3. Lease (Concessão)

Os endereços IP não são atribuídos permanentemente. Eles são emprestados por um tempo determinado. Quando o prazo expira, o cliente pode renovar ou liberar o IP.

4. Reserva de IP

É possível vincular um IP fixo a um MAC address específico. Assim, um dispositivo sempre recebe o mesmo endereço IP, mesmo com o DHCP ativo.

Exemplo:

Uma impressora de rede pode ter sempre o mesmo IP para facilitar o acesso.

5. Escopos e Exclusões

O administrador pode excluir IPs do pool para reservar endereços a servidores e equipamentos com IP fixo, evitando conflitos na rede.

6. Relay Agent

Quando há mais de uma sub-rede, o relay agent encaminha as solicitações DHCP entre as redes, permitindo que um único servidor DHCP atenda a toda a infraestrutura.

DNS — Domain Name System

Sistema hierárquico que converte nomes de domínio (como google.com) em endereços IP (como 142.250.79.110), permitindo que os usuários acessem sites e serviços pelo nome, sem precisar memorizar números.

Funciona como a "agenda telefônica" da internet. Utiliza a porta UDP/TCP 53.

7. Como funciona a resolução DNS

Quando um usuário digita um endereço, o processo de resolução segue estas etapas:

  1. O sistema verifica o cache local do dispositivo
  2. Se não encontrar, consulta o resolver recursivo (geralmente do provedor de internet)
  3. O resolver pergunta ao servidor raiz, que indica o servidor TLD responsável (.com, .br etc.)
  4. O TLD aponta para o servidor autoritativo do domínio
  5. O autoritativo retorna o IP; o resolver armazena em cache e entrega ao cliente

8. Tipos de Registros DNS

Os registros DNS definem como o servidor responde às consultas:

  • A — mapeia domínio para IPv4
  • AAAA — mapeia domínio para IPv6
  • CNAME — apelido que aponta para outro domínio
  • MX — indica o servidor de e-mail do domínio
  • NS — aponta os servidores de nomes autoritativos
  • PTR — resolução reversa: IP → nome de domínio
  • TXT — texto livre, usado para SPF, DKIM e verificações

9. Cache DNS

As respostas DNS são armazenadas por um tempo definido chamado TTL (Time to Live). Isso evita consultas repetidas e agiliza o acesso a domínios já visitados.

10. DNS Reverso

Enquanto o DNS converte nome → IP, o DNS reverso faz o contrário: IP → nome. É utilizado em logs de servidores e na verificação de autenticidade de e-mails.

11. DNSSEC

Extensão de segurança do DNS que assina digitalmente os registros, evitando ataques de falsificação conhecidos como DNS poisoning (envenenamento de cache).

12. Zonas Primária e Secundária

  • Zona primária — servidor autoritativo, onde os registros são criados e editados
  • Zona secundária — cópia de backup que recebe os dados via transferência de zona

Resumo Final

DHCP e DNS trabalham juntos: o DHCP atribui o endereço IP ao dispositivo e já entrega o endereço do servidor DNS junto. A partir daí, o DNS resolve os nomes para que a navegação aconteça.

  • DHCP resolve: "qual é o meu IP na rede?"
  • DNS resolve: "qual é o IP do site que quero acessar?"

Em uma frase: o DHCP configura o dispositivo na rede e o DNS garante que ele encontre os destinos pelo nome.
