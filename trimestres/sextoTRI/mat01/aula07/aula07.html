Aplicação da GPO e suas Particularidades

A GPO (Group Policy Object) é um conjunto de configurações aplicadas automaticamente a usuários e computadores dentro de um domínio do Active Directory. Ela permite que o administrador defina regras de segurança, restrições e configurações de ambiente de forma centralizada, sem precisar configurar cada máquina individualmente.

As GPOs são uma das ferramentas mais poderosas do Active Directory e estão presentes em praticamente toda infraestrutura de TI corporativa baseada em Windows.

1. O que é uma GPO

Uma GPO é um objeto do Active Directory que contém um conjunto de políticas. Cada política define uma configuração específica que será aplicada aos computadores ou usuários do domínio.

  • Configurações de segurança
  • Restrições de acesso
  • Configurações de software
  • Configurações de ambiente de trabalho
  • Scripts de inicialização e encerramento

Exemplo:

Uma GPO chamada "Política de Senhas" pode exigir que todos os usuários do domínio criem senhas com no mínimo 8 caracteres, letras maiúsculas, números e símbolos.

2. Onde as GPOs são aplicadas

As GPOs podem ser vinculadas a três níveis do Active Directory:

  • Site — aplicada a todos os computadores de um local físico da rede
  • Domínio — aplicada a todos os usuários e computadores do domínio
  • Unidade Organizacional (OU) — aplicada apenas aos objetos dentro daquela OU específica

Exemplo:

A empresa tem três setores: RH, TI e Financeiro. Uma GPO vinculada ao domínio inteiro bloqueia o uso de pen drives em todos os computadores. Já uma GPO vinculada apenas à OU de TI libera o uso de pen drives somente para os técnicos.

3. Ordem de Aplicação das GPOs

Quando existem várias GPOs, elas são aplicadas em uma ordem específica, chamada de LSDOU:

  1. L — Local: políticas definidas localmente no próprio computador
  2. S — Site: políticas vinculadas ao site do Active Directory
  3. D — Domain: políticas vinculadas ao domínio
  4. OU — Organizational Unit: políticas vinculadas à unidade organizacional

A última política aplicada tem prioridade. Ou seja, a GPO da OU sobrescreve as anteriores em caso de conflito.

Exemplo:

O domínio define papel de parede padrão para todos. A OU da diretoria tem uma GPO com papel de parede diferente. Os diretores verão o papel de parede da OU, pois ela foi aplicada por último e tem prioridade.

4. GPO de Computador x GPO de Usuário

As configurações dentro de uma GPO são divididas em duas seções:

  • Configurações do Computador — aplicadas quando o computador é iniciado, independentemente de quem faz login
  • Configurações do Usuário — aplicadas quando o usuário faz login, independentemente de qual computador está usando

Exemplo:

A configuração de firewall é feita em "Configurações do Computador", pois precisa estar ativa antes mesmo de alguém fazer login. Já a configuração de papel de parede corporativo é feita em "Configurações do Usuário", pois segue o funcionário em qualquer computador que ele use.

5. Herança de GPO

As GPOs seguem o princípio de herança: OUs filhas herdam automaticamente as GPOs das OUs pai e do domínio.

Exemplo:

O domínio tem uma GPO que bloqueia o Painel de Controle. A OU "Vendas" está dentro do domínio, então todos os funcionários de vendas também terão o Painel de Controle bloqueado, mesmo sem uma GPO específica na OU de Vendas.

6. Bloquear Herança

É possível bloquear a herança em uma OU específica, impedindo que ela receba GPOs dos níveis superiores.

Exemplo:

A OU de TI tem a herança bloqueada. Mesmo que o domínio bloqueie o acesso ao CMD, os técnicos de TI continuam com acesso, pois a OU deles não herda as políticas do domínio.

7. Forçar GPO (Enforced)

A opção Enforced (Forçar) garante que uma GPO seja aplicada em todos os níveis abaixo, mesmo que a herança esteja bloqueada. Ela não pode ser sobrescrita por GPOs de níveis inferiores.

Exemplo:

O domínio tem uma GPO de segurança marcada como Enforced, exigindo antivírus em todos os computadores. Mesmo que a OU de TI bloqueie a herança, essa política de antivírus ainda será aplicada em todas as máquinas.

8. Filtragem de Segurança

Por padrão, uma GPO se aplica a todos os objetos da OU onde está vinculada. Com a filtragem de segurança, é possível aplicar a GPO apenas para usuários ou computadores específicos.

Exemplo:

A OU do RH tem 10 funcionários. Uma GPO de restrição de internet deve ser aplicada apenas a 3 estagiários. Com a filtragem de segurança, a GPO é configurada para afetar somente o grupo "Estagiários RH", sem impactar os demais funcionários da OU.

9. Quando as GPOs são aplicadas

As GPOs são aplicadas nos seguintes momentos:

  • Na inicialização do computador (configurações de computador)
  • No login do usuário (configurações de usuário)
  • A cada 90 a 120 minutos automaticamente em segundo plano
  • Imediatamente ao executar o comando gpupdate /force no CMD

Exemplo:

O administrador cria uma nova GPO bloqueando o acesso ao YouTube. Para não esperar o próximo ciclo automático, ele executa o comando gpupdate /force nos computadores, aplicando a política imediatamente.

10. Precedência entre GPOs

Quando duas GPOs conflitam, a que tem número de ordem menor na lista tem maior prioridade. O administrador pode reorganizar a ordem das GPOs no console.

Exemplo:

A OU de Vendas tem duas GPOs: a GPO-1 define papel de parede azul e a GPO-2 define papel de parede cinza. Como a GPO-1 tem prioridade maior, todos os usuários de Vendas verão o papel de parede azul.

11. GPO e Scripts

As GPOs podem executar scripts automaticamente em momentos específicos:

  • Inicialização do computador — script roda antes do login
  • Encerramento do computador — script roda ao desligar
  • Logon do usuário — script roda quando o usuário entra
  • Logoff do usuário — script roda quando o usuário sai

Exemplo:

Ao fazer login, um script de logon conecta automaticamente o drive de rede do setor do usuário. Ao fazer logoff, outro script desconecta o drive, mantendo a rede organizada.

12. Ferramentas para Gerenciar GPOs

O Windows Server oferece ferramentas específicas para criar e gerenciar GPOs:

  • GPMC (Group Policy Management Console) — console principal para criar, editar, vincular e monitorar GPOs
  • GPME (Group Policy Management Editor) — editor de configurações dentro de uma GPO
  • gpupdate /force — comando para forçar a aplicação imediata das GPOs
  • gpresult /r — comando que mostra quais GPOs estão sendo aplicadas no computador atual
  • RSoP (Resultant Set of Policy) — ferramenta que simula e exibe o resultado final das políticas aplicadas a um usuário ou computador

Resumo Final

A GPO é uma ferramenta essencial do Active Directory que permite aplicar configurações e políticas de forma automática e centralizada em toda a rede. Suas principais particularidades são:

  • Pode ser vinculada a Site, Domínio ou OU
  • Segue a ordem de aplicação LSDOU
  • Divide configurações entre Computador e Usuário
  • Funciona com herança entre OUs
  • Permite bloquear herança ou forçar aplicação com Enforced
  • Permite filtragem por grupo de segurança
  • É aplicada automaticamente a cada 90 minutos ou via gpupdate /force
  • Suporta execução de scripts em logon, logoff, inicialização e encerramento

Em uma frase: a GPO é o mecanismo que permite ao administrador controlar o comportamento de toda a rede a partir de um único ponto, de forma automática, organizada e segura.
