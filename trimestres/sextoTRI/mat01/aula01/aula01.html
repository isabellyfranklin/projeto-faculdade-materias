Como instalar um sistema operacional em máquina virtual

1. O que é um Sistema Operacional?

O Sistema Operacional é o software responsável por gerenciar todos os recursos do computador e servir como intermediário entre o usuário e o hardware. Sem ele, o usuário não conseguiria utilizar o computador de forma prática.

Principais funções:

  • Gerenciar memória RAM e processador
  • Controlar dispositivos de entrada e saída
  • Gerenciar arquivos e pastas
  • Fornecer segurança
  • Permitir a execução de programas

2. Componentes do SO

O SO atua como camada entre: Usuário → Aplicações → Sistema Operacional → Hardware

GUI (Interface Gráfica)

O usuário interage por janelas, ícones e menus. Exemplos: Windows, Ubuntu, macOS.

Shell (Linha de Comando)

Interface baseada em comandos digitados. Exemplos: CMD, PowerShell, Terminal Linux.

3. Sistema Operacional de Rede (SOR)

Projetado para ambientes com vários computadores em rede, normalmente instalado em um servidor. Permite compartilhamento e gerenciamento de recursos.

Recursos do SOR:

  • Compartilhamento de arquivos e impressoras
  • Controle de usuários e permissões
  • Banco de dados e hospedagem de sites
  • Segurança e armazenamento centralizados
  • Gerenciamento de discos (RAID)

4. Classificação dos Sistemas Operacionais

Sistemas Livres

Código-fonte aberto, geralmente gratuitos e personalizáveis. Exemplos: Linux, Ubuntu, FreeBSD.

Sistemas Proprietários

Exigem licença de uso, código fechado, suporte oficial e maior compatibilidade. Exemplo: Windows Server 2019.

5. Windows Server 2019

Principal SOR da Microsoft. Possui quatro versões:

  • Essentials: pequenas empresas, até 25 usuários, sem virtualização
  • Standard: médio porte, virtualização limitada
  • Datacenter: grandes empresas, virtualização ilimitada
  • Hyper-V Server: focado exclusivamente em virtualização

6. Máquina Virtual (VM)

Software que simula um computador completo dentro de outro, com SO, memória, processador, disco e rede virtualizados. Permite testar sistemas sem modificar o computador principal.

Conceitos importantes:

  • Host: o computador físico
  • Guest: o SO instalado dentro da VM
  • Hipervisor: software que cria e gerencia as VMs

7. Tipos de Hipervisor

Tipo 1 — Bare Metal

Funciona direto no hardware. Mais rápido e eficiente. Usado em servidores e data centers.

Estrutura: Hardware → Hipervisor → VM

Tipo 2 — Hospedado

Funciona sobre um SO já instalado. Fácil de instalar, ideal para estudos, porém com menor desempenho.

Estrutura: Hardware → SO → Hipervisor → VM

Exemplo: Windows → VirtualBox → Windows Server

8. Como instalar um SO em uma VM

  1. Instalar o software de virtualização (ex: VirtualBox)
  2. Baixar a imagem ISO do sistema operacional
  3. Criar a VM definindo nome, RAM, processador e disco
  4. Selecionar a ISO como mídia de instalação
  5. Ligar a VM e seguir o assistente de instalação
  6. Configurar a rede da VM

9. Modos de Rede da VM

  • NAT: acessa internet, comunica com o host, não visível na rede externa
  • Rede Somente Host: sem internet, comunica com host e outras VMs, ideal para laboratórios
  • Rede em Ponte (Bridge): acessa internet, visível para toda a rede, simula um computador real

10. Vantagens da Virtualização

  • Testar sistemas sem afetar o computador principal
  • Criar laboratórios de estudo
  • Simular redes empresariais
  • Executar vários SOs no mesmo computador
  • Economizar hardware
  • Facilitar backups e recuperação
