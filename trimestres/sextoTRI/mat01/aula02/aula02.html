Como adicionar serviços no Server 2019 e implementar servidor em Shell

1. Introdução à Segurança de Redes

A segurança da informação é um dos aspectos mais importantes dentro de uma empresa. Apenas instalar um firewall não é suficiente — toda a infraestrutura de TI precisa funcionar corretamente, incluindo:

  • Redes de computadores
  • Servidores
  • DNS (Sistema de Nomes de Domínio)
  • Switches e Roteadores
  • Sistemas Operacionais

Para melhorar a segurança, utiliza-se uma rede baseada em domínio, onde apenas usuários autorizados acessam determinados recursos.

2. O que é o Active Directory Domain Services (ADDS)?

O ADDS é um serviço do Windows Server que funciona como um banco de dados centralizado, armazenando e organizando informações sobre todos os recursos da rede:

  • Usuários;
  • Impressoras;
  • Computadores;
  • >Pastas compartilhadas;
  • Servidores;
  • Grupos de usuários.

Exemplo prático

Em uma rede doméstica, instalar uma impressora exige configurar computador por computador. Com o Active Directory, o administrador configura apenas uma vez e a impressora é instalada automaticamente para todos os usuários autorizados.

3. Autenticação e Domínio

Quando um usuário entra na rede, passa pelo processo de autenticação (logon). O Active Directory é responsável por verificar usuário e senha, confirmar permissões e liberar acesso apenas aos recursos autorizados.

O Domínio é o limite administrativo e de segurança da rede. Após o login, o usuário acessa somente o que o administrador definiu.

4. Instalação do ADDS

  1. Abrir o Gerenciador do Servidor e clicar em "Adicionar funções e recursos"
  2. Selecionar "Instalação baseada em função ou recurso"
  3. Escolher o servidor DC-01
  4. Marcar Active Directory Domain Services e clicar em "Adicionar Recursos"
  5. Marcar DNS Server (necessário para localizar recursos do domínio)
  6. Marcar "Reiniciar automaticamente se necessário" e clicar em Instalar

5. Promovendo o Servidor para Controlador de Domínio

Após instalar o ADDS, o servidor precisa ser promovido. No Gerenciador do Servidor, clique na bandeira de notificação e selecione a opção para promover o servidor. Em seguida:

  • Selecionar "Adicionar uma nova floresta"
  • Definir o domínio: Senac.local (deve começar com letra e conter um ponto)
  • Definir nível funcional: Windows Server 2019
  • Também será necessário criar uma senha para o domínio.
  • Configurações DNS
  • >Durante a instalação pode surgir uma mensagem de aviso relacionada ao DNS.
  • >Isso é normal e não impede a continuidade da instalação.
  • Nome NetBIOS
  • >Definir:
  • SENAC
  • >Esse nome será usado em várias operações internas do domínio.
  • Caminhos dos Arquivos: O sistema exibirá onde serão armazenados: Banco de dados do AD; Logs; Arquivos SYSVOL. Normalmente mantém-se os caminhos padrão.
  • Finalização da Instalação O sistema executará verificações de pré-requisitos.
  • Se tudo estiver correto: Clique em Instalar, Ao final, o servidor se tornará o Controlador de Domínio da rede.

6. Domínio x Work,group

Workgroup (Grupo de Trabalho)

Usado em residências ou pequenas redes. Administração individual, pouca segurança, cada computador possui usuários próprios e configurações feitas máquina por máquina.

Domínio

Usado em empresas. Administração centralizada, maior segurança, controle de acesso, políticas de grupo e gerenciamento de centenas ou milhares de computadores.

7. Inserindo um Cliente no Domínio

Após criar o domínio, adicione os computadores clientes seguindo os passos:

  1. Pressionar Windows + R e digitar sysdm.cpl
  2. Acessar Nome do Computador → Alterar
  3. Selecionar Domínio e digitar Senac.local
  4. Informar usuário Administrador e senha Senac@123
  5. Reiniciar o computador

Após isso, o cliente passará a receber políticas de segurança e poderá acessar os recursos centralizados do servidor AD.

8. Resumo para Prova

  • ADDS: gerencia usuários, computadores e recursos da rede
  • Active Directory: banco de dados centralizado da rede
  • Domínio: limite administrativo e de segurança
  • DNS: localiza recursos dentro da rede
  • Controlador de Domínio: servidor que gerencia autenticação e permissões
  • Workgroup: rede simples e descentralizada
  • Para ingressar no domínio: sysdm.cpl
  • Exemplo de domínio: Senac.local
  • Exemplo de NetBIOS: SENAC
